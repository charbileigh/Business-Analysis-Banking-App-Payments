# Banking App Business Analysis Case Study: Beneficiaries & Payments

## Overview
This repository is a Business Analysis case study for a fictional retail banking mobile app ("MzansiBank").
The scope covers the end-to-end flow for:
1) Adding a beneficiary
2) Making a payment to a beneficiary
3) Handling common payment failures

The goal is to demonstrate BA deliverables that are testable, traceable, and aligned to Agile delivery.

## What this project demonstrates
- Stakeholder & scope definition
- To-Be process design (payments journey)
- Epics, user stories, and Gherkin acceptance criteria
- Business rules & non-functional requirements (NFRs)
- Data dictionary for key entities
- QA-ready artifacts (test scenarios + traceability matrix)
- Risks, issues, and decisions log

## Feature Scope
### In Scope
- Add / view / manage beneficiaries
- Make payment to a beneficiary
- OTP (one-time PIN) step-up verification for risky payments
- Failure handling (insufficient funds, invalid OTP, cut-off time)

### Out of Scope
- Card management, loans, and investments
- International payments
- Merchant payments (QR / bill payments)
